وثيقة الاتفاق الرسمية بين فريق AI Brain وفريق Knowledge & Platform.

أساسي

نوعان من الـ APIs — افهم الفرق أولاً

النوع الأول — Internal APIs (نحن نبنيها)

  • نكتبها بـ FastAPI داخل مشروع ReTour
  • هي عقد التواصل بين Alpha و Beta
  • مثال: POST /api/v1/packages/generate
  • الـ Frontend لا يتكلم مع الـ Agent مباشرة — يمر عبر API

النوع الثاني — External APIs (خدمات خارجية)

  • خدمات من شركات أخرى — لا نبنيها
  • مثال: Weather API · Google Maps · OpenStreetMap
  • Beta يستهلكها ويغلّفها داخل Context API
  • Alpha لا يعرف تفاصيل Weather API أبداً

❌ خطأ — لا نريد هذا

Agent (Alpha) │ ▼ ChromaDB (Beta) /* Alpha مربوط بطريقة التخزين */

Agent (Alpha) │ ▼ Knowledge API (Beta) │ ▼ ChromaDB (Beta)

Context API — الطقس والمسافات والمواعيد

Team Beta يبنيها — Alpha يستهلكها Week 5

Beta مسؤول عن جلب المعلومات الخارجية وتغليفها. Alpha لا يعرف تفاصيل Weather API أو Maps API.

❌ خطأ

Agent → Weather API (مباشرة)

✅ صح

Agent → Context API → Weather + Maps
POST /api/v1/context
Owner: Team Beta

Alpha يقول: "أريد حالة الأماكن في عمان بتاريخ معيّن." Beta يجمع من Weather + Maps + Opening Hours.

▼ Request — Alpha يرسل

الاجتماع

ماذا يقول كل فريق؟ — الالتزام الرسمي

هذه هي الـ API Agreement — يجب الموافقة عليها قبل البدء

Team Beta يقول:

"نحن نوفر لكم هذه الـ APIs:"

  • POST /knowledge/search — للحصول على المعرفة السياحية
  • POST /context — للحصول على الطقس والمسافات والمواعيد
  • POST /smes/recommend — للحصول على مرشد مناسب
  • POST /smes — Dashboard لإدارة المرشدين
  • React UI + Innovation Hub
  • ChromaDB + Tourism Data + SME Database

Team Alpha يقول:

"نحن نبني:"

  • POST /packages/generate — توليد الباقة
  • ReTour Tourism Agent (وكيل واحد)
  • LLM Manager + RAG Orchestrator
  • Decision Rules + Package Validator
  • نستخدم APIs الخاصة بكم — لا ندخل ChromaDB أو Weather مباشرة
